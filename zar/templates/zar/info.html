Zoom Attendance Report Aggregator

What does it do?

This is a tool for aggregating your zoom attendance reports into a single report that is actually useful to you. If you didn't already know, you can export a report of the "usage," from your zoom meetings (aka who attended and for how long). That's pretty nifty, but who really wants to sort through all that data, filter out students' weird inaccurate zoom names, and compile a longitudinal report of students' attendance over time?

This website, that's who.

Will this work if I use the same meet topic for all my Zooms?

Yes! That is actually where this tool shines the most. If you are a teacher who has been using the same meet topic to meet with different groups of students, it would be a nightmare for you to try to disaggregate all your meet reports from zoom and make any use out of them. This tool can actually dynamically group meetings based on the overlap of students between different meeting instances.

In plainer terms, that means that if you used the meet topic "Smith Music" to meet with every homeroom in every grade, this tool will group together each homeroom and print them separately on the final report.

Will this work if my students use incorrect names in Zoom?

Yes! Again, this is one of the intended use cases of this tool. It will agressively try to match the student's zoom name with the student's real profile. First it will try an exact full name match. If that fails, it performs an approximate name match. If that fails to return a sufficiently confdent match, it will try to match the first name to a first name in the group it presumes to currently be looking at.

For example, if all the attendees of the meeting currently being parsed are in Mrs. Smith's homeroom and we encounter a student named "Jimmy Fortnite Rules Forever," The program will first break that atrocious name into four separate parts: ['Jimmy', 'Fortnite'...]. Then, it will try to find a first-name-match within Mrs. Smith's homeroom for each word. If there is only one "Jimmy" in Mrs. Smith's homeroom, we have a match.

Suffice it to say, the script will match almost any name that a human could have. Key word "almost." After all is said and done, you will be brought to a skippable screen where you can manually match any stragglers. The system also gets smarter over time, so the straggler-to-name conversion that you input is saved and potentially used by the next person so they don't have to do as much manual matching.

Is this Safe?

I used a web framework called Django that includes all of the security features of any modern website. Cross site scripting protection, cross site request forgery tokens, SQL injection protection, clickjacking protection, encryption, password hashing, and more are all implemented here.

Plus, you cannot make an account or do anything here unless you have been verified to have access to an @empacad.org email address, and no, students cannot gain access with their accounts either.
